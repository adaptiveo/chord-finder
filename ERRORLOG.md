# Error Log

Журнал ошибок и их решений. Сортировка: новые сверху.

## Формат записи

```markdown
### ERR-XXX: Краткое описание
- **Дата:** YYYY-MM-DD
- **Проект:** Название проекта
- **Контекст:** Что делалось
- **Ошибка:** Текст ошибки / симптомы
- **Причина:** Почему произошло
- **Решение:** Конкретные шаги исправления
- **Профилактика:** Как избежать в будущем
- **Теги:** #build #deploy #dependency
```

### ERR-001: PyInstaller кэширует старый код
- **Дата:** 2026-04-21
- **Проект:** checkinn.gui
- **Контекст:** Сборка exe после исправления FilePicker
- **Ошибка:** В exe всё ещё старая ошибка "Unknown control: FilePicker"
- **Причина:** PyInstaller использует кэш build/, dist/, .pyc
- **Решение:** Удалить перед сборкой: `rm -rf build dist __pycache__ *.pyc`
- **Профилактика:** Всегда чистить кэш при странных ошибках в exe
- **Теги:** #build #pyinstaller #cache

### ERR-002: Pandas не найден в собранном exe
- **Дата:** 2026-04-22
- **Проект:** checkinn.gui
- **Контекст:** Повторная обработка в exe
- **Ошибка:** "No module named 'pandas'"
- **Причина:** pandas используется только в `process_inns_list()`, PyInstaller не видит импорт
- **Решение:** Добавить `--hidden-import pandas` в build.py
- **Профилактика:** Проверять импорты в новых методах перед сборкой
- **Теги:** #build #pyinstaller #pandas

### ERR-003: KeyError 'inn' при загрузке Excel
- **Дата:** 2026-04-22
- **Проект:** checkinn.gui
- **Контекст:** Загрузка входного файла
- **Ошибка:** `Ошибка: 'inn'`
- **Причина:** Excel содержит числовые ИНН с `.0` (например, `1234567890.0`)
- **Решение:** Убирать `.0`: `if inn.endswith('.0'): inn = inn[:-2]`
- **Профилактика:** Нормализовать данные из Excel перед валидацией
- **Теги:** #excel #data-validation #parsing

### ERR-004: Входной файл перезаписывается результатами
- **Дата:** 2026-04-22
- **Проект:** checkinn.gui
- **Контекст:** Пользователь выбрал один файл для входа и выхода
- **Ошибка:** Исходные данные потеряны
- **Причина:** Нет валидации на совпадение путей
- **Решение:** Добавить проверку `Path(input).resolve() == Path(output).resolve()`
- **Профилактика:** Всегда проверять, что вход != выход
- **Теги:** #ui #validation #data-loss

### ERR-005: FilePicker не работает в Flet 0.80+
- **Дата:** 2026-04-21
- **Проект:** checkinn.gui
- **Контекст:** Миграция с Flet 0.6x на 0.80+
- **Ошибка:** "Unknown control: FilePicker"
- **Причина:** API изменился: FilePicker теперь в `page.services` вместо `page.overlay`
- **Решение:** Использовать `page.services.append(file_picker)`
- **Профилактика:** Проверять breaking changes в changelog библиотеки
- **Теги:** #flet #ui #breaking-changes

### ERR-006: SUS2 парсится как SUS2SUS (двойная замена)
- **Дата:** 2026-04-22
- **Проект:** chord-finder
- **Контекст:** Добавление аккордов sus2 (Fsus2, Gsus2)
- **Ошибка:** "Chord 'FSUS2' not found" - нормализация давала FSUS2SUS
- **Причина:** `.replace("SUS2", "~2~").replace("~", "SUS")` → SUS2SUS
- **Решение:** Использовать ASCII placeholder ``, `` без конфликтов
- **Профилактика:** Не использовать символы-разделители, которые встречаются в целевых строках
- **Теги:** #kotlin #string-replace #parsing

### ERR-007: MethodTooLargeException при добавлении аккордов
- **Дата:** 2026-04-22
- **Проект:** chord-finder
- **Контекст:** Добавление 476 аккордов из pianochord.org
- **Ошибка:** `MethodTooLargeException: Method exceeds 64KB limit`
- **Причина:** Kotlin inline-данные в коде превысили лимит метода JVM
- **Решение:** Вынести аккорды в JSON assets, загружать runtime через `context.assets.open("chords.json")`
- **Профилактика:** Большие датасеты (>500 записей) — в ресурсы, не в код
- **Теги:** #kotlin #jvm #method-size #json

### ERR-008: Gadd11 показывается как Gadd9
- **Дата:** 2026-04-22
- **Проект:** chord-finder
- **Контекст:** Исправление add-аккордов после добавления поддержки ADD9
- **Ошибка:** Ввод Gadd11, показывает Gadd9 (ноты C E D вместо C E F)
- **Причина:** Все ADD-варианты нормализовались в один ключ `ADD` с формулой add9
- **Решение:** Создать отдельные записи ADD2 [0,2,4,7], ADD [0,4,7,14], ADD11 [0,4,7,17]
- **Профилактика:** Разные варианты аккордов = разные ключи в базе
- **Теги:** #kotlin #chords #data-model
