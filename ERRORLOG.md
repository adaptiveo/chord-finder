### ERR-011: Дублирование пальцев при отрисовке барре
- **Дата:** 2026-04-26
- **Проект:** chord-finder
- **Контекст:** Проверка отображения аккорда G на укулеле
- **Ошибка:** На диаграмме укулеле для аккорда G показаны:
  - Барре на 2 ладу через струны C, E, A с цифрой "1"
  - Отдельный палец "2" на струне A, 2 лад (избыточно)
  - Палец "3" на струне E, 3 лад
- **Причина:** 
  - Код отрисовки сначала рисует все пальцы, затем поверх рисует барре
  - Пальцы, попадающие на барре, не исключаются из отрисовки
  - В данных аккорда G для укулеле есть отдельный fret для струны A на 2 ладу, который покрывается барре
- **Решение:** 
  - Добавить проверку `isCoveredByBarre` перед отрисовкой каждого пальца
  - Пропускать отрисовку пальца, если его лад и струна попадают в диапазон барре
- **Профилактика:** 
  - При отрисовке диаграмм с барре всегда проверять пересечение с пальцами
  - Барре рисуется поверх пальцев только визуально, но пальцы под барре не должны отображаться
- **Теги:** #ui #compose #chord-diagram #barre #ukulele
- **Статус:** ✅ Исправлено (2026-04-26)

### ERR-012: Барре создавалось для несмежных струн
- **Дата:** 2026-04-26
- **Проект:** chord-finder
- **Контекст:** Исправление ERR-011, проверка генерации гитарных аккордов
- **Ошибка:** Функция `findBarres()` создавала барре для всех струн с одинаковым ладом, даже если они не соседние. Например, если струны 1 и 6 имели лад 1, создавалось барре через все струны 1-6.
- **Причина:** Отсутствовала проверка на смежность струн при создании барре.
- **Решение:** 
  - Добавить проверку `isContiguous` — барре создаётся только если струны соседние (разница индексов = 1)
  - Использовать `zipWithNext()` для проверки последовательности струн
- **Профилактика:** 
  - Барре возможно только на соседних струнах — это физическое ограничение игры на гитаре/укулеле
  - Всегда проверять смежность при группировке элементов
- **Теги:** #chord-generator #barre #algorithm #guitar #ukulele
- **Статус:** ✅ Исправлено (2026-04-26)

### ERR-014: Приглушённые струны не отображались на диаграмме
- **Дата:** 2026-04-26
- **Проект:** chord-finder
- **Контекст:** Полная ревизия проекта по запросу пользователя
- **Ошибка:** Приглушённые струны (fret = -1, обозначаются как "X" на диаграмме) не отображались. Аккорд D для гитары должен отображаться как "xx0232" (струны 6 и 5 приглушены), но X-маркеры не были видны.
- **Причина:** В `ChordDiagram.kt` (строка 266) условие `if (fret.fret <= 0) return@forEach` находилось ДО блока `when { fret.fret == -1 => ... }`, что блокировало отрисовку X-маркеров.
- **Решение:** 
  - Вынести отрисовку X-маркеров в отдельный цикл ДО общей проверки пропуска
  - Сначала отрисовываются все X (fret = -1), затем обрабатываются остальные позиции
- **Профилактика:** 
  - Всегда проверять порядок условий — специальные случаи (-1) должны обрабатываться до общих условий пропуска (<= 0)
  - Добавить тесты на отрисовку приглушённых струн
- **Теги:** #ui #compose #chord-diagram #rendering #guitar
- **Статус:** ✅ Исправлено (2026-04-26)

### ERR-015: Инверсии вместо корневых позиций аккордов
- **Дата:** 2026-04-26
- **Проект:** chord-finder
- **Контекст:** Аккорд D для гитары отображался неправильно — использовалась инверсия D/A вместо корневой позиции D
- **Ошибка:** 
  - Аккорд D генерировался с открытой струной A (струна 5), создавая инверсию D/A
  - Ожидаемая позиция: xx0232 (струны 6,5 приглушены, струна 4 открыта D)
  - Фактическая позиция: xA0232 (струна 5 открыта A, инверсия)
- **Причина:** 
  - В `tryOpenPosition()` (строка 243) условие `stringIndex > lowestRootString` проверяло струны с большим индексом
  - В гитаре больший индекс = более тонкая/высокая струна
  - Струны с меньшим индексом (более низкие/толстые) не отключались правильно
  - Для D: `lowestRootString = 2` (струна D), струны 0 (E) и 1 (A) имели `stringIndex < 2`, но условие не срабатывало
- **Решение:** 
  - Заменить `stringIndex > lowestRootString` на `stringIndex < lowestRootString`
  - Струны с меньшим индексом (ниже корня) теперь отключаются (fret = -1)
  - Расширен диапазон поиска ладов с 1..4 до 1..5 для лучших позиций
- **Профилактика:** 
  - Помнить: в гитаре индекс 0 = самая низкая струна (E), индекс 5 = самая высокая (e)
  - Для предотвращения инверсий отключать струны ниже корня
  - Тестировать все базовые аккорды (C, D, E, F, G, A, Am, Em, Dm)
- **Теги:** #chord-generator #guitar #inversion #root-position #algorithm
- **Статус:** ✅ Исправлено (2026-04-26)

### ERR-013: Неверная генерация аккорда D для гитары
- **Дата:** 2026-04-26
- **Проект:** chord-finder
- **Контекст:** Пользователь сообщил, что аккорд D для гитары указан неправильно
- **Ошибка:** При генерации аккорда D алгоритм выбирал неверные позиции струн. Например, открытые струны добавлялись в опции даже если их нота не входила в аккорд.
- **Причина:** В `ChordGenerator.generateChordPosition()` (строка 139) условие `if (note in chordNotes || fret == 0)` добавляло все открытые струны (лад 0) независимо от того, входят ли их ноты в аккорд.
- **Решение:** 
  - Изменено условие на `if (note in chordNotes)` — открытые струны добавляются только если их нота входит в аккорд
  - Это гарантирует, что алгоритм не будет выбирать неверные открытые струны
- **Профилактика:** 
  - Всегда проверять, что ноты открытых струн входят в аккорд
  - Тестировать генерацию на всех базовых аккордах (C, D, E, F, G, A, Am, Em, Dm)
- **Теги:** #chord-generator #guitar #data-validation
- **Статус:** ✅ Исправлено (2026-04-26)

### ERR-010: Неверные данные аппликатур для гитары и укулеле
- **Дата:** 2026-04-22
- **Проект:** chord-finder
- **Контекст:** Проверка отображения аккорда C на укулеле
- **Ошибка:** Для аккорда C на укулеле показываются все открытые струны `[0,0,0,0]`, что неверно
- **Причина:** 
  - Источник данных pianochord.org содержит **только пианино-аппликатуры**
  - Данные для гитары и укулеле либо отсутствуют, либо сгенерированы автоматически с ошибками
  - В JSON для укулеле часто указаны все нули (открытые струны) вместо реальных позиций
- **Решение:** Требуется заменить данные на корректные из специализированных источников:
  - Гитара: ultimate-guitar.com, chordie.com, или специализированные API
  - Укулеле: ukulele-tabs.com, uke-chords.com
- **Профилактика:** 
  - Всегда проверять источник данных перед импортом
  - Для каждого инструмента использовать специализированные ресурсы
  - Валидировать данные (открытые струны для всех струн — подозрительно)
- **Теги:** #data-quality #validation #guitar #ukulele
- **Статус:** ✅ Исправлено (2026-04-24)
- **Решение:** 
  1. Исправлена инверсия струн в `ChordDiagramCanvas` (строка 266: `strings - fret.string` → `fret.string - 1`)
  2. Исправлена инверсия в отрисовке барре (строки 308-309)
  3. `ChordGenerator.generateChordPosition()` теперь проверяет наличие ВСЕХ нот аккорда в позиции
  4. Открытые струны используются только если их нота входит в аккорд
  5. Струны без подходящих нот помечаются как «не играют» (-1)
- **Результат:** Аккорды для гитары и укулеле генерируются динамически с правильными нотами

### ERR-009: Dropdown не закрывается после выбора элемента
- **Дата:** 2026-04-22
- **Проект:** chord-finder
- **Контекст:** Реализация BACK-001 (Autocomplete dropdown)
- **Ошибка:** 
  1. При выборе "G" из списка dropdown открывается снова
  2. При выборе **первого элемента** списка dropdown не закрывался
- **Причина:** 
  1. LaunchedEffect(query) реагирует на изменение query и пересчитывает suggestions, открывая dropdown
  2. `onValueChange` в OutlinedTextField тоже устанавливал `expanded = true`, создавая конфликт с LaunchedEffect
- **Решение:** 
  1. Добавить флаг `justSelected` — при выборе устанавливать флаг, а в LaunchedEffect пропускать обновление с задержкой 300ms
  2. Убрать установку `expanded` из `onValueChange` — пусть только LaunchedEffect управляет состоянием dropdown
- **Профилактика:** 
  - При изменении state из UI всегда учитывать side-effects от других observers (LaunchedEffect)
  - Не дублировать управление одним и тем же state'ом из разных мест
- **Теги:** #ui #compose #state-management #bug
- **Статус:** ✅ Исправлено и протестировано на устройстве

### ERR-006: SUS2 парсится как SUS2SUS (двойная замена)
- **Дата:** 2026-04-22
- **Проект:** chord-finder
- **Контекст:** Добавление аккордов sus2 (Fsus2, Gsus2)
- **Ошибка:** "Chord 'FSUS2' not found" - нормализация давала FSUS2SUS
- **Причина:** `.replace("SUS2", "~2~").replace("~", "SUS")` → SUS2SUS
- **Решение:** Использовать ASCII placeholder ``, `` без конфликтов
- **Профилактика:** Не использовать символы-разделители, которые встречаются в целевых строках
- **Теги:** #kotlin #string-replace #parsing

### ERR-008: Gadd11 показывается как Gadd9
- **Дата:** 2026-04-22
- **Проект:** chord-finder
- **Контекст:** Исправление add-аккордов после добавления поддержки ADD9
- **Ошибка:** Ввод Gadd11, показывает Gadd9 (ноты C E D вместо C E F)
- **Причина:** Все ADD-варианты нормализовались в один ключ `ADD` с формулой add9
- **Решение:** Создать отдельные записи ADD2 [0,2,4,7], ADD [0,4,7,14], ADD11 [0,4,7,17]
- **Профилактика:** Разные варианты аккордов = разные ключи в базе
- **Теги:** #kotlin #chords #data-model
