# Nota Bene

Критически важные заметки, инсайты и правила, выработанные из ошибок.

## Документирование

- **NB-1:** Лучшее время для документирования - сразу после решения проблемы, пока контекст свежий
- **NB-2:** Конкретные пути и команды > общие описания
- **NB-3:** Каждая ошибка в ERRORLOG должна иметь "Профилактику"

## Сборка EXE

- **NB-4:** PyInstaller кэширует агрессивно - всегда чисти `build/`, `dist/`, `__pycache__` при странных ошибках
- **NB-5:** `--collect-all pandas` добавляет ~12MB тестов и лишнего кода
- **NB-6:** Playwright браузер должен быть включен через `--add-data` для portable-режима

## Flet UI

- **NB-7:** Все обработчики `on_click` в Flet 0.80+ должны быть `async def`
- **NB-8:** FilePicker добавляется в `page.services`, не в `page.overlay` (изменение API 0.80+)
- **NB-9:** `pick_files()` возвращает список, `save_file()` возвращает строку - не путать типы

## Работа с Excel

- **NB-10:** Excel хранит числа как float - ИНН могут иметь `.0` на конце
- **NB-11:** Пустые строки в Excel считываются как `None` или пустые кортежи
- **NB-12:** Первый столбец может быть не единственным - проверять `len(row)`

## Git

- **NB-13:** `.gitignore` должен включать `__pycache__/`, `*.pyc`, `dist/`, `build/`
- **NB-14:** Бинарники (exe) не коммитятся в git, только в релизах

## Архитектура

- **NB-15:** Разделяй core (логика) и ui (интерфейс) - никаких импортов UI в core
- **NB-16:** Wrapper - единственная точка контакта между core и UI
- **NB-17:** Callbacks для прогресса лучше sync, UI обновления - через `page.update()`

## Отрисовка аккордов

- **NB-18:** При отрисовке диаграмм с барре исключать пальцы, попадающие в диапазон барре. Иначе будет дублирование (палец + барре на той же позиции)
- **NB-19:** Барре возможно только на соседних струнах. При генерации барре проверять смежность струн (`zipWithNext`), иначе будут неверные барре через все струны
- **NB-20:** Открытые струны при генерации аккордов включать только если их нота входит в аккорд. Иначе алгоритм выберет неверные позиции
